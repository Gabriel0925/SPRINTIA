Sprintia 4.3.1

Optimisation

Date

24 Mai 2026

Taille

-- Mo

Résumé

Sprintia 4.3.1 apporte de nombreuses optimisations ce qui permet à Sprintia de devenir plus rapide, plus fiable et moins exposé à des bugs qui pourraient gâcher l'expérience.

De plus, Sprintia 4.3.1 intègre un tout nouveau footer permettant d'accéder à certaines pages clés en un clic, au lieu de 3 ou 4 auparavant.

Patch note

Nouvelles fonctionnalités

Améliorations

  • Désormais dans l'algorithme de Récupération vous avez accès uniquement à votre historique des 7 derniers jours. Pour avoir accès à votre historique complet il faudra désormais cliquer sur le bouton "Tout voir" qui vous ammènera à une page dédiée pour ça.
  • L'algorithme d'estimation VMA ne peut pas prédire une VMA avec une course libre supérieur à 20 km, auparavant c'était 30 km or on n'estime pas une VMA sur de si longue distance. Déjà 20 km étant énorme pour estimer une VMA ! Je vous conseille de privéligiez les autres options des tests VMA comme le demi-cooper ou le cooper ou le luc léger. Le mode libre est conçue si vous n'avez pas fais de test vous pouvez l'estimer mais attention vous perdez grandement en précision.

Correction des bugs et optimisations

  • Optimisation du CSS pour le responsive de la page nommé "Historique des versions".
  • Correction d'un bug lors de la modification d'un entraînement les données de transpiration estimée et d'hydratation estimée pouvait être enregistré avant des données comme la distance,... Or la distance était censé se sauvegardé avant pour qu'elle apparaisse avant lors de la consultation du détail de votre entraînement.
  • Optimisation du calcul des dates : il y a désormais une nouvelle fonction que je peux appeller sur n'importe quelle page. Cette fonction centralise la logique pour trouver une date par exemple pour trouver J-7, J-31,...
  • Correction d'un bug majeur du logo dynamique dans le détail d'un entraînement avec une datas spécifique ce qui bloquait la page. L'utilisateur ne pouvait donc plus modifier son entrainement, plus le supprimer,...
  • Optimisation du code de l'algorithme d'estimation VMA : la fonction qui gère le test VMA que vous avez choisi a été optimisé passant d'une longue structure if/else à l'usage de dico. Il y a de nombreuses autres optimisations, en globalité le code de cette algorithme passe de 127 à 104 lignes pour les mêmes fonctionnalités.
  • Optimisation du code de l'algorithme de temps de récupération, le code est désormais plus maintenable et optimisé. Passage de 59 lignes à 44 ce qui est toujours ça de pris.
  • Optimisation du code de l'algorithme de l'estimation VO₂max, le code est désormais bien plus maintenable et optimisé. Auparavant, en Janvier je faisais une succession de plus de 5 if/else if/else (pour trouver dans quelle zone le user se trouvait par rapport à sa tranche d'âge et à l'estimation de son VO₂max) ce qui est illisible, inmaintenable,... Cette algorithme faisait auparavant 185 lignes, désormais il fait 94 lignes !
  • Dans l'estimation du VO₂max, si les inputs était vide et que vous changiez de Sexe l'algorithme se lançait et vous disais que vous n'avez pas remplis les champs or l'algorithme n'est pas censé se lancer si les inputs sont vidés, désormais ce bug est résolu.
  • Ajout d'un fonction global pour les messages d'erreur dans l'algorithme de Niveau de course. Ce changement a été réalisé pour les futures fonctionnalités qui sera disponible dans les futures versions.
  • Lors d'une erreur dans l'algorithme de Niveau de course une erreur s'affiche dans le formulaire. Désormais, l'erreur s'affiche en rouge. Auparavant, c'était de la couleur du thème choisi or en UX/UI c'est pas très bien de faire ça, c'est pour ça que c'est corrigé dans Sprintia 4.3.1.
