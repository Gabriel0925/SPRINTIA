Résumé

Dans la dernière mise à jour, j'ai reposé des bases solides pour le futur de SPRINTIA avec la version nommé STRIDE I. Maintenant avec SPRINTIA 5.1, j'apporte plein de nouvelles fonctionnalités en commençant par SPRINTIA Briefing.

SPRINTIA Briefing est une extansion du JRM Coach, cette fonctionnalité a pour but de créer une liaison entre les modèles IA grand public (comme Gemini, Claude, Chat GPT, Vibe, Grok,...). Cette liaison vous permet d'appronfondir l'analyse de votre récupération, de votre charge d'entraînement et plus encore.

Patch note

Nouvelles fonctionnalités

  • SPRINTIA Briefing est une extansion du JRM Coach, cette fonctionnalité a pour but de créer une liaison entre les modèles IA grand public (comme Gemini, Claude, Chat GPT, Vibe, Grok,...). Cette liaison vous permet d'appronfondir l'analyse de votre récupération, de votre charge d'entraînement et plus encore.
  • SPRINTIA Briefing vous permet donc de poser des questions sur votre récupération, sur votre prochaine séance ou même pour construire un programme d'entraînement pour réaliser votre objectif. Les possibilités sont infinies avec SPRINTIA Briefing, la seule limite c'est vos idées.
  • Dans les paramètres de SPRINTIA Briefing, vous pouvez choisir quelle IA vous allez utilisez pour analyser vos données. Donc si vous préférez ChatGPT à Vibe, vous avez juste à séléctionner ChatGPT dans les paramètres.
  • Dans les paramètres de SPRINTIA Briefing, en plus d'avoir le choix de votre IA favorite, vous avez également quelques paramètres comme la prise en compte de la personnalité de votre coach ou encore les niveaux d'analyse pour adapter la complexité du vocabulaire de l'IA car tout le monde n'aiment pas les jargons techniques tandis que d'autres adorent.
  • Dans la page "À propos de SPRINTIA Briefing" je vous explique quelle données sont partagées aux IA,...

Correction des bugs et optimisations

  • Les données que vous enregistrez après un entraînement est désormais cohérent ce qui facilite l'export de données de SPRINTIA à un autre service. Auparavant la vitesse moyenne pouvait être enregistrée comme ça : "30" or c'est un nombre, donc désormais en toute logique j'enregistre ça 30. De plus, des datas restait en null si vous n'aviez rien saisi dans ce input, désormais si vous n'avez rien saisi dans ce input rien n'est enregistré dans la base de donnée. Ce changement a été répliqué dans l'import de fichier TCX.
  • Correction d'un texte quand il n'y a pas de donnée dans l'outil "Récupération". Ce texte est désormais plus précis et il informe mieux l'utilisateur sur ce qu'il doit faire.
  • Modification de la durée des animations du logo dynamique.
  • Modification de la durée d'affichage du logo dynamique.
  • Correction d'un bug du logo de l'icône SPRINTIA sur PC.
  • Unification des animations css lors d'un clic sur un élément de l'interface. Certains éléments avaient été oublié dans SPRINTIA 5.
  • Ajustement du design du container du toggle pour respecter la règle UI des arrondis.
