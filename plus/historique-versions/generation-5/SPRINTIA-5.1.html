Résumé

Dans la dernière mise à jour, j'ai reposé des bases solides pour le futur de SPRINTIA avec la version nommée STRIDE I. Maintenant avec SPRINTIA 5.1, j'apporte plein de nouvelles fonctionnalités en commençant par SPRINTIA Briefing.

SPRINTIA Briefing est une extension du JRM Coach, cette fonctionnalité a pour but de créer un raccourci entre les modèles IA grand public (comme Gemini, Claude, ChatGPT, Vibe, Grok,...). Ce raccourci vous permet d'approfondir l'analyse de votre récupération, de votre charge d'entraînement et plus encore.

Contributeur

SPRINTIA 5.1 a été dévellopé avec Isaac Lestage ! C'est la première fois que j'ouvre SPRINTIA à un ami. Pendant que je m'occupais de l'import TCX et GPX, Isaac a entièrement conçu l'affichage de la carte de votre tracé GPS dans le détail de votre entraînement.

N'hésitez pas à découvrir ses projets sur son GitHub.

Patch note

Nouvelles fonctionnalités

  • SPRINTIA Briefing est une extension du JRM Coach, cette fonctionnalité a pour but de créer un raccourci entre les modèles IA grand public (comme Gemini, Claude, ChatGPT, Vibe, Grok,...). Ce raccourci vous permet d'approfondir l'analyse de votre récupération, de votre charge d'entraînement, de votre entraînement et plus encore.
  • SPRINTIA Briefing intègre une toute nouvelle interface de discussion. Grâce au nouveau champ de texte, vous pouvez poser directement vos propres questions à votre IA favorite (jusqu'à 500 caractères). Qu'il s'agisse de comprendre une baisse de forme, d'analyser un RPE anormalement haut sur vos dernières sorties, ou de faire un plan d'entraînement, SPRINTIA Briefing croise votre prompt avec les données que vous avez enregistrées sur SPRINTIA. Les possibilités sont infinies ! La seule limite est votre imagination.
  • Dans les paramètres de SPRINTIA Briefing, vous pouvez choisir quelle IA vous allez utiliser pour analyser vos données. Donc si vous préférez ChatGPT à Vibe, vous avez juste à sélectionner ChatGPT dans les paramètres.
  • Dans les paramètres de SPRINTIA Briefing, en plus d'avoir le choix de votre IA favorite, vous avez également quelques paramètres comme la prise en compte de la personnalité de votre coach ou encore les niveaux d'analyse pour adapter la complexité du vocabulaire de l'IA, car tout le monde n'aime pas le jargon technique tandis que d'autres adorent.
  • Dans la page "À propos de SPRINTIA Briefing" je vous explique quelles données sont "partagées" aux IA,...
  • Une toute nouvelle fonctionnalité de nettoyage de données fait son apparition ! SPRINTIA n'ayant pas été conçue pour accumuler un historique d'entraînement lourd, vous pouvez désormais libérer facilement de la place dans le cache de votre navigateur et donc dans le stockage de votre appareil. Vous pouvez choisir de conserver uniquement les données des 30 derniers jours, 90 derniers jours et même 365 derniers jours.
  • Désormais lorsque que vous importez votre fichier TCX SPRINTIA récupère les points GPS de votre entraînement.
  • SPRINTIA 5.1 apporte enfin l'import GPX de votre entraînement et au passage ça sauvegarde directement les points GPS de votre entraînement.
  • Si vous importez le fichier TCX ou GPX de votre entraînement, SPRINTIA va savoir créer une carte dans le détail de votre entraînement pour que vous consultez le tracé GPS de votre entraînement (cette fonctionnalité a été développée par un ami (Isaac Lestage), n'hésitez pas à découvrir son GitHub ).

Améliorations

  • Le JRM Coach arbore un nouveau design dégradé avec différentes nuances de bleu pour aller de pair avec le style graphique de SPRINTIA Briefing.
  • Désormais, dans la page de profil, il n'y a que les données de votre profil ; le téléchargement de vos données, la restauration de vos données et la réinitialisation de SPRINTIA se trouvent désormais dans les paramètres.
  • Étant donné que le nombre de paramètres s'agrandit, j'ai rajouté des noms pour chaque section, ce qui permet d'améliorer l'UX de SPRINTIA.
  • Nouveau design de la carte pour configurer son profil. De plus, vous pouvez enfin ignorer cette invitation à configurer votre profil si vous le souhaitez et la carte disparaîtra comme par magie !
  • La cohérence des titres des pages d'import de données a été revue pour être plus cohérent entre eux.

Correction des bugs et optimisations

  • Les données que vous enregistrez après un entraînement sont désormais cohérentes, ce qui facilite l'export de données de SPRINTIA vers un autre service. Auparavant, la vitesse moyenne pouvait être enregistrée comme ça : "30" alors qu'il s'agit d'un nombre, donc désormais en toute logique j'enregistre ça : 30. De plus, des datas restaient en null si vous n'aviez rien saisi dans cet input ; désormais, si vous n'avez rien saisi dans cet input, rien n'est enregistré dans la base de données. Ce changement a été répliqué dans l'import de fichier TCX.
  • Correction d'un texte quand il n'y a pas de donnée dans l'outil "Récupération". Ce texte est désormais plus précis et il informe mieux l'utilisateur sur ce qu'il doit faire.
  • Modification de la durée des animations du logo dynamique.
  • Modification de la durée d'affichage du logo dynamique.
  • Correction d'un bug du logo de l'icône SPRINTIA sur PC.
  • Unification des animations CSS lors d'un clic sur un élément de l'interface. Certains éléments avaient été oubliés dans SPRINTIA 5.
  • Ajustement du design du conteneur du toggle pour respecter la règle UI des arrondis.
  • Optimisation du chargement de la page de détail d'entraînement.
  • Correction de plusieurs fautes d'orthographe.
  • Les bibliothèques "EmailJS" et "Papaparse" sont désormais intégrées localement à l'application, ce qui améliore l'indépendance de SPRINTIA.
  • Correction d'un bug mineur dans la configuration du JRM Coach.
  • Correction d'un bug majeur dans la restauration de vos données. J'enregistrais toutes les datas de récupération du fichier à l'ID 1, étonnamment ça ne provoquait pas de bug mais je ne devais pas faire comme ça !
  • Correction d'un bug mineur lors de la restauration de vos données. Si le code plantait, le bouton n'était pas fonctionnel pour réessayer d'importer un fichier JSON.
  • Les fonctionnalités de téléchargement de données, de restauration de données et même de réinitialisation de SPRINTIA sont toutes optimisées, elles sont désormais bien plus rapides ! Le code est également bien plus robuste et maintenable ! C'est une très grosse avancée du point de vue du code.
  • Correction d'un bug mineur dans la structure de la base de données.
  • Grosse optimisation pour l'import de fichier TCX. La fonctionnalité est désormais bien plus robuste et bien plus rapide.
  • L'arborescence pour la fonctionnalité d'import de données a été totalement revu pour améliorer la maintenabilité du code.
  • Amélioration des performances du chargement de la page d'import de données.
  • Correction d'un bug sur le composant "segmented-button", c'était un bug de cohérence lié au BFC Cache, il est désormais corrigé. Le comportement de ce composant a été légèrement modifié au tactile et à la souris.
